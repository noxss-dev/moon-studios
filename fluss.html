| Cakey is a «selection model»          |
| that selects commonly                 |
| a random image from its               |
| library («The Cakey Library»).        |
| The original founder (nuggetqq)       |
| refused to use match/case so          |
| it used if/elseif/.../else            |
| instead. The Cakey Library has        |
| random images from the Internet,      |
| and each one is related to            |
| a "trigger" (shape/animal/...).       |
| The trigger itself does one,          |
| specific thing, so to combine         |
| adjectives and other grammar stuff    |
| it uses its own parser integrated     |
| in Cakey. The dynamic of the trigger  |
| is inspired by Google Nano Banana,    |
| which, to be honest, IT'S NOT dev     |
| friendly. Let me explain: if ya wanna |
| make an AI that uses Nano Banana YOU  |
| are gonna make the parser, YOU are    |
| gonna use an unknown technique lost   |
| by an antique egpytian. And so on.    |
| No hate to Google though, the design  |
| of this app is Materialize, which is  |
| made by a Google employee, because    |
| Google announced Material Design on   |
| Android. So, yeah.                    |
